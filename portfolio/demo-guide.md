# 데모 영상 녹화 가이드 (Demo Recording Guide)

> 이 프로젝트를 **5~6분**(핵심 컷)으로 보여주는 녹화 대본·동선. 채용담당/보안엔지니어가 본다고 가정.
> 축 = **차별점 3개**: ① 크로스클라우드 attack-path ② AI 능동조사(챗봇 탈출) ③ RAG 지식베이스.
> 화면 동선 원본은 [docs/console-manual.md §5](../docs/console-manual.md).

---

## 0. 녹화 전 준비 (5분)

- **하드 새로고침(Ctrl+Shift+R)** — 옛 번들 캐시 제거. 가능하면 **시크릿창**(확장·알림 없음).
- **해상도 1920×1080 · 브라우저 줌 100%** · OS/브라우저 알림 끄기 · 방해금지.
- **녹화 툴**: OBS Studio(무료·고품질) / Windows `Win+G` 게임바 / Loom(간편·웹).
- **미리 1회 리허설** — 클릭 순서·로딩 지점 파악. 로딩 긴 곳은 나중에 컷 편집.
- **로그인은 미리** 해두고 시작하거나(빠름), 로그인 장면을 짧게 포함(기업 SSO 어필).
- 마이크 내레이션 or 자막 — **담백하게**(과장 금지, 사실 위주).
- ⚠️ 전부 **합성 데이터**(가짜 PII·시크릿)라 노출 걱정 없음. 계정ID는 데모 계정이라 무방.

## 1. 오프닝 — 무엇을·왜 (20~30초)

**화면**: 대시보드(로그인 후) 또는 타이틀 슬라이드.
**말할 것(요지)**:
> "멀티클라우드 — **AWS는 워크로드, Azure Entra ID는 신원** — 환경의 보안 위험을 한 화면에 통합하고, 그 위에 **에이전틱 AI**로 조사·설명·조치까지 잇는 CNAPP 보안 플랫폼입니다. 차별점은 세 가지: **① AWS→Azure를 넘는 크로스클라우드 공격경로 추적, ② AI가 스스로 API를 호출해 증거를 모으는 능동조사, ③ 벡터 검색 기반 RAG 설명**입니다."

## 2. 로그인 · 대시보드 — 첫인상 + 전체 조망 (40초)

- (선택) **SSO 로그인**: "Microsoft Entra ID로 로그인" → Cognito → 대시보드. "기업 SSO(Entra→Cognito→SPA)로 인증하고, 역할(APPROVER/viewer)은 Entra 그룹으로 판정됩니다."
- **대시보드**: AWS/Azure **Secure Score**, KPI(전체·open·Critical·공격경로), **기둥별 open**.
- 마지막에 **🛑 크로스클라우드 attack-path 배너**를 가리키며: "가장 위험한 건 이 배너 — 클릭하면 공격 경로가 나옵니다."

## 3. 크로스클라우드 Attack-path — 차별점 ① (60초)

- 배너 클릭 → **그래프**. **AWS 노드(주황)·Azure 노드(파랑)**를 나누고, **경계를 넘는 빨강 엣지**를 마우스로 짚으며:
  > "왼쪽 AWS 워크로드에서 시작한 침해가, 이 **빨강 크로스클라우드 엣지**를 타고 오른쪽 **Azure Entra 신원**까지 번집니다."
- 아래 **공격 서사** 낭독:
  > "product 취약 이미지로 침투 → order의 과도한 IRSA와 **평문 Azure 시크릿** 획득 → member의 **공개 S3에서 PII 탈취** → 그 자격증명으로 **Entra 과도권한 앱을 장악**해 디렉터리 전체 통제."
- 강조: "단일 클라우드 도구는 이 **경계 횡단**을 못 봅니다. 이게 멀티클라우드 CNAPP의 핵심."

## 4. AI 능동조사 (Evidence) — 차별점 ② · ★크라운주얼 (90초)

- Findings 목록 → 상위 finding 클릭 → **🔎 능동조사(Evidence) 탭**.
- **판정 히어로** 짚으며: "AI가 **N회 read-only API를 스스로 호출**해 증거를 모으고 판정했습니다."
- 아래로 스크롤하며 순서대로:
  - **① 트리아지 게이트**(비용 통제 — 중요한 것만 조사)
  - **② 가설**(AI가 세운 공격 가설)
  - **③ tool 호출 타임라인** ← 여기서 잠깐 멈춤: "`s3:GetBucketPolicy`, `iam:SimulatePrincipalPolicy`… **AI가 어떤 API를 부를지 스스로 골라** 실제로 호출한 기록입니다."
  - **④ 판정 + 지식베이스 근거**
- **핵심 멘트**:
  > "이게 이 프로젝트의 심장입니다. **질문에 답만 하는 챗봇이 아니라, LLM이 스스로 도구(read-only API)를 호출해 증거를 수집**합니다. 저는 이걸 '챗봇 탈출'이라 부릅니다."
- (선택·approver) **🤖 AI 재조사** 버튼 클릭: "실시간으로 다시 조사시킬 수도 있습니다."
- 💡 **판정이 `refuted`로 나와도 당황 말 것** — "실제 증거로 조사해서 대상이 진짜 취약하지 않으면 정직하게 refuted를 냅니다. **고무도장이 아니라 진짜 조사**한다는 증거예요."

## 5. AI 어시스턴트 (RAG) — 차별점 ③ (40초)

- **🤖 AI 어시스턴트(/chat)** → 추천 질문 클릭 or "S3 버킷이 공개되면 왜 위험하고 어떻게 조치해?"
- 답변 뜨면 **근거(refs)** 짚으며:
  > "질문을 **Titan으로 임베딩 → pgvector 코사인 유사도 검색 → Bedrock이 그 지식으로 답변**. 답 아래 **근거 control**이 실제로 검색된 지식베이스 청크입니다. 환각이 아니라 근거 기반."

## 6. AI·시스템 관측 — "진짜 돈다" 증명 (30초)

- **AI·시스템(/system)**: 파이프라인 스트립 · AI 모델 3종 · **Bedrock 사용량 24h(실 CloudWatch)** · **RAG 지식베이스 통계(pgvector)**.
  > "목업이 아니라 **실제 Bedrock 호출·실제 pgvector**가 돕니다. 사용량·토큰까지 관측됩니다."

## 7. 조치(HITL) + 감사 — 안전장치 (30초)

- **조치**: 승인 대기 큐 → (approver로) 승인 → "Step Functions로 실행되고 **감사로그에 불변 기록(S3 Object Lock)**됩니다."
- **감사로그** 잠깐: "모든 판정·조치·스캔이 시간순 불변 기록. **read-only first, 변경은 승인 경로로만**."

## 8. 클로징 — 아키텍처 + 기술스택 (20~30초)

**화면**: 아키텍처 다이어그램 슬라이드(또는 README 그림) + AI·시스템 파이프라인.
> "타깃 앱은 **agentless**로 스캔만 되고, 콘솔은 저장소를 읽기만 합니다. 배포는 **GitOps(ArgoCD)**, 인프라는 **전부 Terraform(IaC)**로 재현 가능하고, **무료 크레딧 안에서 프로덕션급 아키텍처**를 증명했습니다. AWS + Azure Entra, Bedrock 멀티에이전트 + RAG, EKS·pgvector — 이상입니다."

## 길이 전략
- **메인 5~6분**(위 전체) = 포폴 본편.
- **짧은 2~3분 컷**(채용담당 초두 이탈 대비): 오프닝 → attack-path → **Evidence 능동조사** → 클로징. (RAG·관측·조치는 생략)
- 유튜브/드라이브 **비공개 링크** + 챕터 타임스탬프(위 섹션 제목) 달면 스캔하기 좋음.

## 녹화 팁
- 마우스 **천천히**, 클릭 직전 0.5초 멈춤. 커서로 짚어가며 설명.
- 로딩·빈 순간은 **컷 편집**으로 제거(템포 유지).
- 화면 텍스트가 작으면 **브라우저 줌 110~125%** 로 키워 녹화.
- 첫 테이크 완벽 필요 없음 — 섹션별로 끊어 찍고 이어붙이기.
- 배경음악 넣을 거면 아주 작게(내레이션 우선).

## 녹화 전 체크리스트
- [ ] `deploy.ps1 apply`로 전 레이어 살아있음 + 타깃 파드 Running + ArgoCD Synced
- [ ] Entra SAML 식별자 = 현 Cognito Pool(로그인 됨)
- [ ] 콘솔 하드 새로고침(최신 번들) · Evidence 탭·/chat·/system 정상 렌더 확인
- [ ] (원하면) attack-path·Evidence가 있는 finding 미리 열어둬 동선 매끄럽게
- [ ] 녹화 끝나면 **비용 규율**: `deploy.ps1 -Action destroy -AutoApprove` (+ monitoring은 진우 별도)
